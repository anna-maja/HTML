...åsså inser jag att jag är lite osäker på vad "barn" och "syskon" innebär i det här sammanhanget...

Universal selector

- alla element i dokumentet.
- asterisk

Type selector

- elementnamn
- t ex "h1, h2, h3"

Class selector

- klasselement
- påverkar dem med ett visst värde efter punkt
- t ex ".note"
- t ex "p.note" = alla .note-element som ligger i p

ID selector

- baseras på hashtag
- alla element med text som är exakt som efter hashtaggen

child selector

- element som är direkt barn till annat element
- li>a påverkar alla a-element som är barn till li, men inga andra a-element.
...jag fattar nog inte vad barn är...

Descendant selector

- elemtnt som är underliggande ett annat element, men itne barn till elementet.
- p a = påverkar alla a-element som ligger i p-element.
- a-elementen kan ha andra element före sig.

Adjacent sibling selector

- element som är syskon till andra element.
- t ex h1+p = påverkar första p efter alla h1, men inte andra p-element.

General sibling selector

- element som är syskon till varandra, även om det inte är i anslutning till varann.
- h1~p = om du har två p-element som är syskon till h1, påverkas båda.
